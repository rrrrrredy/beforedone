# v1.0.0 manual release evidence

This record closes the local/manual release boundary for BeforeDone v1.0.0.
Repository Actions were intentionally disabled before publishing, so this
release does not claim GitHub OIDC build provenance.

## Immutable inputs

- Source commit and public `main` at release time:
  `b0b17adc083e34607d51cffec0f2a37bcdd3f346`.
- Public tag: `v1.0.0`, pointing to the same commit. GitHub Release creation
  produced a lightweight tag; the published tag was not rewritten afterward.
- Published at: `2026-07-21T06:13:06Z`.
- Release: <https://github.com/rrrrrredy/beforedone/releases/tag/v1.0.0>
- Pages source: classic `gh-pages:/`, commit
  `f0a600b0d0c000365482404882476b3c8dbaf586`.

## Public installability audit

- The public Windows amd64 archive was downloaded, extracted, and run on
  Windows x64. `beforedone version` reported `1.0.0`; the public Adapter Kit
  fixtures passed 3/3.
- `go install github.com/rrrrrredy/beforedone/cmd/beforedone@v1.0.0` succeeded
  with GOBIN, GOPATH, module cache, and build cache isolated under a new D-drive
  audit directory. The installed binary reported `1.0.0` and passed the same
  fixtures. The Go proxy currently resolves `@latest` to `v1.0.0`.
- The tagged Marketplace source resolves `plugins/beforedone`; manifest version,
  two Skills, hooks, and Windows/Unix wrappers are present. With the public
  Windows binary on a process-local PATH, the PowerShell Hook persisted a real
  `SessionStart` event and fail-closed a `Stop` event when project configuration
  was missing.
- Both tagged standalone `SKILL.md` URLs return HTTP 200. The public skills.sh
  pages for `verify-before-done` and `investigate-agent-incident` also return
  HTTP 200. The Skills-only Directory bundle contains the two Skills and their
  metadata byte-for-byte after newline normalization, and contains no hooks.
- The official Codex `skill-installer` helper installed both exact `v1.0.0`
  paths into an isolated D-drive destination. The command exited 0 and all four
  installed files matched the Git blobs at release commit
  `b0b17adc083e34607d51cffec0f2a37bcdd3f346` byte-for-byte. The helper and test
  did not write the user's global Skills directory.
- The Directory package verifier now checks the complete ZIP file tree, declared
  sizes, regular-file modes, compression method, and every packaged byte against
  the LF-normalized repository source. It accepted the published ZIP; a fresh
  rebuild reproduced its exact SHA-256. It rejected separate variants with a
  same-length Skill content change, a non-regular file mode, and an unexpected
  compression method.
- The exact published Skills-only ZIP installed and enabled through a minimal
  local marketplace in a fresh isolated Codex home. Codex's model-visible skill
  inventory contained `beforedone:verify-before-done` and
  `beforedone:investigate-agent-incident`. The source and installed cache each
  contained the same five files with matching hashes and declared zero Hook,
  MCP, or App components. The user's global Codex state remained unchanged.
- CLI smoke runs for all five positive Directory review fixtures matched their
  expected behavior: first-run init/doctor/check/receipt PASS; stale-receipt
  detection before refresh; missing-verifier INCONCLUSIVE; a FAIL incident with
  JSON and HTML artifacts; and replay analyze plus dry-run verify with zero
  external verifier commands.
- Codex CLI `0.145.0-alpha.18` added the public Git marketplace and installed
  `beforedone@beforedone` v1.0.0 into an isolated `CODEX_HOME`. The installed
  manifest, two Skills, seven lifecycle event registrations, and Windows/Unix
  hook scripts matched the marketplace source. The user's global Codex config
  and plugin directory remained unchanged.

The audit did not write the user's global Codex or Skills directories.

## Explicitly outside this evidence boundary

This record does not claim native execution of the macOS, Linux, or Windows
arm64 archives. This host has no WSL distribution, Docker/Podman runtime, Linux
user-mode emulator, or macOS environment, so cross-compilation and archive
inspection are not presented as runtime proof. It also does not claim OpenAI
Directory approval, final media, YouTube publication, or a live Product Hunt
listing. Those remain visible in the launch control room instead of being
inferred from build success.
